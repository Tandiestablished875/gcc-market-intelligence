# Contributing to GCC Market Intelligence

Thank you for considering a contribution. This skill is a community resource for founders entering the Gulf, and corrections / additions from people on the ground make it materially better for the next founder who picks it up.

## What we welcome

**High-confidence corrections.** If a regulatory threshold, program ticket size, family business holding, or fact in any file is wrong or stale, please open a pull request with the corrected information AND a verifiable source. Cite the URL, the date you accessed it, and the specific page or section.

**Additions backed by public information.** New named programs, events, system integrators, family branches, sovereign-fund vehicles, soft-landing tracks, or regulatory frameworks — all welcome. Each addition should follow the existing file structure (TL;DR → details → "what outside founders should know" → "what NOT to do" → "verify before committing capital").

**Sector-specific addenda.** If you have deep operational experience in a vertical (defense procurement, halal certification, fintech sandbox specifics, healthcare procurement, oil & gas supply chain), please open an issue describing the addendum you'd like to contribute — happy to discuss the right home for it.

**Regulatory updates.** When a regulation moves, flag the file and the date in an issue. PRs welcome that update the file with the new state and a "Last verified" date refresh.

## What we won't merge

**Speculation about ruling-family internal dynamics.** Unverified deal rumors, succession speculation, allegations about specific individuals — all out of scope. The skill stays on public, verifiable, professionally-relevant facts.

**Unsourced claims.** "I heard from someone" is not a source. Sources need to be public, verifiable, and named.

**Promotional content.** Don't add your own company / service / consulting firm to the skill files as a recommended provider. The skill names organizations because they're materially significant in the market (sovereign funds, programs, SIs, events), not as recommendations.

**Confidential information.** If you learned it from an NDA-protected engagement or under professional confidence, it doesn't belong here. The skill is intentionally built only from public information.

## Style guide

- **TL;DR first.** Every file opens with a 60-second briefing.
- **Numbers need sources.** Every claim about a ticket size, quota, threshold, or budget should be sourced (footnote URL or inline reference to the regulator).
- **Cite the regulator, not the consultant.** GASTAT > McKinsey on Saudi statistics; SDAIA > Gartner on PDPL; MoHRE > Forbes on Emiratisation.
- **Tone: direct, operational.** Avoid corporate flavor language ("synergies," "robust solutions," "ecosystem"). The reader is a busy operator.
- **Prose for refusals and caveats.** Bullet points are for tables and parallel lists. Caveats go in prose.
- **Verify before committing capital.** Every file ends with a "Verify" section pointing to official sources for time-sensitive data.

## Pull request process

1. **Open an issue first** for substantial changes — keeps coordination clean.
2. **One file or one logical change per PR.** Easier to review, easier to merge.
3. **Update the "Last verified" date** in any file you edit substantively.
4. **In the PR description:** what changed, why, with sources. If updating a fact, include both the old version and the source supporting the new version.

## Maintainers

This skill was originally built with reference to the [Madinah Tech Cultivator](https://madinahcultivator.com) Cohort 2 founder profile. Contributions from former and current cohort founders, regional advisors, and operators with on-the-ground experience are particularly valued.

## Code of conduct

Be precise. Be kind. Disagree about facts, not people. The Gulf business community is small; reputational respect propagates. The same applies inside this repo.
